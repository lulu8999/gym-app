# 第5课：cat / head / tail / wc（查看文件内容）

## 核心命令

```bash
cat 文件名        # 一次性全看（小文件用）
head 文件名       # 默认看前10行
head -5 文件名    # 看前5行
tail 文件名       # 默认看后10行
tail -5 文件名    # 看后5行
tail -f 文件名    # 实时跟踪（日志用，Ctrl+C退出）
wc 文件名         # 行数、单词数、字节数
wc -l 文件名      # 只看行数
```

## 教学记录

**日期：** 2026-06-13

**用户基础：** 已学到 cp/mv

**本课重点：**
- cat = 全部显示，head = 前几行，tail = 后几行，wc = 统计
- 小文件用 cat，大文件用 head/tail

**用户提问：**
- Q: "etc是什么意思" → etcetera（拉丁语"等等"），/etc 放系统配置文件，类比物业管理办公室
- Q: "passwd呢" → password缩写，/etc/passwd 存用户信息
- Q: "容易出错的点" → 主动询问陷阱，说明注重实操安全

**学习风格观察：**
- 喜欢追问词源和命名含义，不是死记命令
- 重视常见陷阱，会主动问"哪里容易出错"
- 偏好微信上学习，比 CLI 更顺手
- 类比法效果好（大楼比喻、物业管理办公室）

**常见陷阱（已教）：**
1. cat 大文件会刷屏 → Ctrl+C 中断
2. head/tail 默认10行，不是全部
3. tail -f 会卡住等新内容，和 tail -5 完全不同
4. wc 不加 -l 输出三列（行数、单词数、字节数）
