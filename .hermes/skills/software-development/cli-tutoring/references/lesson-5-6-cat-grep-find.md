# 第5-6课：cat/head/tail/wc + grep/find

**日期：** 2026-06-13

## 第5课：看文件内容（cat/head/tail/wc）

### 核心命令
```bash
cat 文件名        # 全部打出来
head -5 文件名    # 前5行
tail -5 文件名    # 后5行
wc -l 文件名      # 只数行数
```

### 用户测验结果：4/5
- ✅ wc三个数字（行数、单词数、字节数）
- ❌ 用 `cat -5` 代替 `head -5`（cat没有行数参数）
- ✅ wc -l 数行数
- ✅ tail -f 会卡住
- ✅ wc不加-l输出三列

### 常见坑
1. `cat` 没有 `-n` 参数选择行数，想看前几行用 `head`
2. `tail -f` 实时跟踪会卡住终端（Ctrl+C退出）
3. 大文件用 `cat` 会刷屏，用 `head`/`tail` 更安全

## 第6课：搜索查找（grep/find）

### 核心命令
```bash
grep "关键词" 文件名        # 在文件内容里找
grep -n "关键词" 文件名     # 带行号
grep -r "关键词" 目录名     # 递归搜目录
grep -i "关键词" 文件名     # 忽略大小写
find 目录 -name "文件名"    # 找文件
find / -name "*.py"         # 全局找
```

### 用户测验结果
**第一轮：2/5** — grep和find严重混淆
- ❌ 用find搜内容、grep搜文件名
- ❌ 参数混用（-name给grep用）

**辨析练习：5/5** — 全对！概念已攻克

**巩固轮：4.5/5** — 大幅进步
- ❌ `grep -r` vs `grep -i` 混淆（第4题少了-r）

### 核心混淆点（已攻克）
| | grep | find |
|--|------|------|
| 找什么 | 文件**内容**里的字 | **文件名** |
| 按名字找 | ❌ | `-name "xxx"` |
| 按内容找 | `"关键词"` | ❌ |

**口诀：grep找内容，find找文件，参数各用各的不串**

### 辨析题设计（效果好）
给5条命令让用户判断对错，比纯实操题更能暴露概念混淆。用户反馈"再来一轮辨析题"说明这种题型有效。
