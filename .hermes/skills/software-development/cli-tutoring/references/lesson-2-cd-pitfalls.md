# cd 命令学习中的典型错误

来源：用户 Lulu 学习过程中的真实错误

## 错误一：cd 和 .. 之间没有空格

**用户输入：**
```bash
cd..
```

**系统报错：**
```
bash: cd..: command not found
```

**原因：**
`cd` 是命令，`..` 是参数，两者必须用空格分开。
系统认为你在试图执行一个叫做 "cd.." 的命令，而这个命令不存在。

**正确写法：**
```bash
cd ..
```

## 错误二：相对路径误用（跨分支）

**情境：** 用户在 `/home`，想去 `/root/.hermes`

**用户输入：**
```bash
cd root/.hermes
```

**系统报错：**
```
bash: cd: root/.hermes: No such file or directory
```

**原因：**
`/home` 和 `/root` 是平级关系（都在 `/` 下面），不是父子关系。
从 `/home` 无法通过相对路径走到 `/root`。

**目录结构：**
```
/                    ← 根目录
├── /home          ← 你在这（没有 root 子目录）
│
└── /root          ← 要去这（平级关系）
    └── .hermes
```

**正确写法：**
```bash
cd /root/.hermes       # 绝对路径
cd ~/.hermes           # 用 ~ 简写
```

## 错误三：返回两级目录计数错误

**情境：** 在 `/root/.hermes/scripts`，想回到 `/root`

**用户输入：**
```bash
cd ..
```

**实际结果：**
到了 `/root/.hermes` （只上了一层）

**原因：**
`/root/.hermes/scripts` → `/root/.hermes` → `/root` 需要上两层

**正确写法：**
```bash
cd ../..               # 上两层
cd /root               # 绝对路径直接跳
```

---

## 概念模糊：绝对路径 vs 相对路径

**用户疑问：**
> "`/root/.hermes` 和 `root/.hermes` 不是一样吗？"

**解释：**

| 路径 | 类型 | 含义 |
|--------|------|-------|
| `/root/.hermes` | 绝对路径 | 从根目录 `/` 开始找，不管你现在哪 |
| `root/.hermes` | 相对路径 | 从当前目录开始找，找当前目录下的 `root` 文件夹 |

**类比：**
- 绝对路径 = 完整地址"江苏省南京市..."
- 相对路径 = "往前走两步，右转"

---

## 总结

**cd 命令易错点检查清单：**
- [ ] `cd` 和参数之间有空格吗？
- [ ] 相对路径不引用时，确认目标是否在当前分支下？
- [ ] 返回多层时，`..` 的个数对吗？
- [ ] 不确定时，用 `pwd` 先看看在哪

**残酷现实：**
> 在 Linux 里，一个空格可以是从"正常执行"到"系统崩溃"的距离。
> 例如 `rm -rf / root` （有空格）vs `rm -rf /root` （无空格）。
