# 新手常见卡壳点及解答

## SSH/终端相关

**Q: 怎么进 VPS？**
A: SSH 连接，如果之前连过，在终端敲：`ssh root@你的IP`。如果是 Windows，用 PowerShell 或 Putty。

**Q: 我怎么"退出来"？**
A: `Ctrl+D` 退出 SSH。如果在 `less` 或 `top` 里，按 `q` 退出。

**Q: 命令卡住了没反应？**
A: `Ctrl+C` 中断当前命令，回到提示符。

**Q: 历史命令怎么找？**
A: 按方向键 ↑ 可以翻之前的命令。

## 文件相关

**Q: 提示 permission denied？**
A: 文件权限不够。大部分时候用 `sudo` 前缀再试一次，如 `sudo cat /var/log/syslog`。

**Q: 文件删了能找回吗？**
A: Linux 命令行没有回收站，`rm` 就是永久删除。慎用。

**Q: `ls -la` 出来的文件列表怎么看？**
A: 每行格式：权限 链接数 所有者 组 大小 日期 文件名。以 `.` 开头的文件是隐藏文件。

**Q: `/` 和 `/root` 有什么区别？**\nA: `/` 是根目录（文件系统最顶层，类比整栋楼），`/root` 是 root 用户的家目录（类比楼里你的房间）。敲 `ls /` 看系统结构，敲 `ls /root` 看自己的文件。

**Q: `ls -la` 第一列的 `-` 和 `d` 是什么意思？**\nA: 第一个字母是**文件类型**：`-` = 普通文件，`d` = 目录，`l` = 软链接。不是 `.`（`.` 开头的文件名是隐藏文件，跟类型无关）。

**Q: 文件路径太长记不住怎么办？**
A: 按 Tab 键自动补全文件名和路径。

## 命令相关

**Q: command not found？**
A: 命令没安装或打错了。检查拼写，或问我装没装。

**Q: 命令输出太多刷屏了？**
A: 在命令后面加 ` | less` 可以翻页看，如 `pm2 logs | less`。

**Q: 管道是什么？**
A: 竖线 `|` 把左边的输出传给右边继续处理。如 `ls | grep .py` = 列出文件，然后过滤出含 .py 的。

## 进程相关

**Q: 怎么知道一个东西在不在跑？**
A: `pm2 list` 看 PM2 管理的。`ps aux | grep 名字` 看所有进程。

**Q: 服务起不来了？**
A: 敲 `pm2 logs xxx --lines 20` 看最后 20 行日志，通常能看出原因。

## 心理建设

**Q: 怕敲错命令搞坏东西？**
A: 在 `/root` 下敲 `ls` `cd` `cat` 看看文件，只读操作坏不了任何东西。不敢确定的时候先问我。
