# 第1课：pwd / ls / ls -la —— 我在哪、有什么文件

## 教学步骤

### 0. 先侦察用户环境

第一课开始前，先确认用户是哪个用户、什么系统：
- 用户让敲 `pwd`，看家目录路径 → 判断是 root 还是普通用户（如 lighthouse）
- 之后敲 `ls` → 看目录是不是空的
- 再敲 `ls -la` → 看隐藏文件和详细信息

### 1. pwd — Print Working Directory

- 显示当前所在目录的绝对路径
- 用户 VPS 通常是 `/root`（root 用户）或 `/home/用户名`（普通用户）
- 这是 Linux 导航的起点

### 2. ls — List files

- 列出当前目录的文件和目录
- 不加参数只显示非隐藏文件
- 提示：新用户家目录可能只有隐藏文件，所以 `ls` 输出可能是空的，很正常

### 3. ls -la — Full details

`-l` = long format（详细信息），`-a` = all（包括隐藏文件）

输出格式逐列讲解：

```
权限        链接 所有者  组      大小  日期        文件名
drwx------  4   user   user    126  Jun 1 22:50  .
-rw-r--r--  1   user   user    376  Oct 23 2025  .bashrc
```

### 4. 权限列详解（最重要）

**第一位 —— 文件类型：**
- `-` = 普通文件
- `d` = 目录
- `l` = 软链接（快捷方式）

**后九位 —— 三组权限（rwx）：**

```
rwx  rwx  rwx
 ↑    ↑    ↑
自己  同组  其他人
```

- `r` = 读
- `w` = 写
- `x` = 执行（对文件=能当成程序跑，对目录=能 cd 进去）
- `-` = 没有该权限

**举例：**
- `-rw-r--r--` = 普通文件，所有者可读写，其他人只能读
- `drwx------` = 目录，只有自己能进，其他人不能碰
- `-rwxr-xr-x` = 可执行程序，自己可读写跑，其他人只能读和跑

**核心概念：配置文件的 x 没有意义**\n- `.bashrc` 是启动时读取的配置，不需要可执行权限\n- 目录必须有 x 才能进入（cd）

### 常见误解（真实用户反馈）

**1. 文件类型第一位：不是 `.`，是 `-`**
- 用户常以为 `.` 表示文件，`d` 表示目录
- 正确：`-` = 普通文件，`d` = 目录，`l` = 软链接
- `.` 开头的文件名是**隐藏文件**，与类型无关

**2. `/`（根目录）与 `/root`（家目录）混淆**
- `/` = 文件系统最顶层（整个系统）
- `/root` = root 用户的个人家目录（类比：整栋楼 vs 你的房间）
- 用户 PR：`ls /` vs `ls /root` 对比实操最有效

### 5. total 20

不是文件数量，是占用的磁盘块数量。1 块=512 字节。total 20 ≈ 10KB。

### 6. 日期列

显示**最后修改时间**（mtime），不是创建时间。Linux 不记录创建时间。

```
.bash_history → Jun 4 09:49  ← 刚才敲的命令刚写进去
.bashrc       → Oct 23 2025  ← 系统初始化时生成的，没改过
```

Linux 的三种时间：
| 时间 | 命令 | 含义 |
|------|------|------|
| mtime | `ls -l` | 内容最后修改 |
| ctime | `ls -lc` | 属性/权限变更 |
| atime | `ls -lu` | 最后读取 |

## 常见发现

用户执行 `ls -la` 后通常会看到：

| 文件 | 说明 |
|------|------|
| `.bashrc` | 登录时自动执行的配置脚本 |
| `.bash_profile` | 登录 shell 的配置文件 |
| `.bash_logout` | 退出时执行的脚本 |
| `.bash_history` | 命令历史记录 |
| `.npm/` | Node.js 包缓存目录 |
| `.openclaw/` | 旧系统配置目录 |

所有以 `.` 开头的文件都是**隐藏文件**，`ls` 不加 `-a` 看不到。

## 教学节奏

```
我发提醒 → 用户说"来了" → 我出三个命令（pwd、ls、ls -la）
       → 用户上 VPS 敲 → 截图/文字告诉我看到了什么
       → 我逐列解释权限列 → 用户提问 → 答疑
       → 让用户试 cat .bashrc 看内容（做延伸）
       → 结束
```

## 延伸问题

用户可能会问：
- "为什么有的文件我只能读写不能执行？" → 配置文件不需要 x
- "total 20 是什么意思？" → 磁盘块数量
- "日期是创建时间吗？" → 是修改时间 mtime
- "除了目录都用 - 表示吗？" → 还有 l（链接）、c（字符设备）、b（块设备）
