# grep vs find 高频混淆专项

## 用户背景
Lulu 在第 6 课（搜索查找）中反复混淆 grep 和 find 的参数，连续三轮测验都在同一类错误上丢分。

**2026-06-14 更新：** 复习时测验 4/5，混淆问题基本解决。剩余错误（grep语法顺序写反）属于新错误模式，不再是经典的 grep/find 参数混淆。

## 混淆历史（已基本解决）

### 错误模式（2026-06-13 记录，2-3轮后已改善）
```
❌ find -n /etc/passwd          # 把 find 当成了 grep
❌ grep ".log" /root            # 想找文件，用了 grep
❌ grep /root -name "*hello"    # 给 grep 加了 find 的 -name 参数
❌ grep -r "error" /var/log → 用户答"要加 -i"  # 混淆 -r 和 -i
```

### 2026-06-14 复习时新错误
```
❌ grep /var/log/syslog "Failed password"   # 顺序反了
✅ grep "Failed password" /var/log/syslog   # 正确：关键词在前，文件在后
```
**根因：** grep 语法顺序（`grep "关键词" 文件`），不是参数混淆。用户理解了 find/grep 职责边界，只是 grep 的参数顺序偶尔会颠倒。

## 终极记忆法（已验证有效）

> 🔍 **grep = 阅读器**（在文件里找字）
> 📂 **find = 侦探**（在目录里找文件）

**口诀**：find 找门牌号，grep 找屋里东西

| | `grep` | `find` |
|--|--------|--------|
| 找什么 | 文件**内容**里的字 | **文件名** |
| 按名字找 | ❌ 不会 | `-name "xxx"` |
| 按内容找 | `"关键词"` | ❌ 不会 |
| 忽略大小写 | `-i` | ❌ |
| 递归搜目录 | `-r` | 默认就是递归 |
| 显示行号 | `-n` | ❌ |
| 按大小找 | ❌ 不会 | `-size +1G` |
| 按类型找 | ❌ 不会 | `-type f`（文件）/ `-type d`（目录）|

## find -type 参数教学记录

用户在复习中对 `-type f` 不理解，需要单独讲解。

**有效类比（食物点菜）：**
- `-type` = "我要什么类型的菜"（文件 vs 目录）
- `-name` = "我要叫什么名字的菜"
- 两个可以组合用，先筛类型再筛名字

**常用类型：**
| 参数 | 找什么 | 记忆 |
|------|--------|------|
| `-type f` | 普通文件 | f = file |
| `-type d` | 目录 | d = directory |
| `-type l` | 软链接 | l = link |

**教学建议：** 用户对参数不理解时，先用类比解释"为什么需要这个参数"，再给用法。
