# 第6课：grep / find（搜索查找）

## 核心命令

```bash
# grep — 在文件内容里搜关键词
grep "关键词" 文件名
grep -i "关键词" 文件名      # 忽略大小写
grep -r "关键词" 目录名      # 递归搜整个目录下所有文件内容
grep -n "关键词" 文件名      # 显示行号

# find — 在目录里找文件
find 目录 -name "文件名"
find / -name "*.mp3"         # 在整个系统找所有mp3
find /root -name "test*"     # 在root下找test开头的文件
find /root -type f -name "*.py"  # 只找文件（排除目录）
find /var -type d -name "log*"   # 只找目录
find /home -type f -size +1G     # 大于1G的文件
find /root -mtime -7             # 7天内修改过的文件
```

## 口诀

> 🔍 **grep = 内容搜索器**（在文件里找字）
> 📂 **find = 文件侦探**（在目录里找文件）
>
> 找文件 → `find -name "xxx"`
> 找内容 → `grep "xxx" 文件名`

## 参数归属速查表

| | `grep` | `find` |
|--|--------|--------|
| 找什么 | 文件**内容**里的字 | **文件名** |
| 按名字找 | ❌ 不会 | `-name "xxx"` |
| 按内容找 | `"关键词"` | ❌ 不会 |
| 忽略大小写 | `-i` | ❌ |
| 递归搜目录 | `-r` | 默认就是递归 |
| 显示行号 | `-n` | ❌ |

## 教学记录

**日期：** 2026-06-13

**用户基础：** 已学 cat/head/tail/wc（第5课）

**本课重点：**
- grep 和 find 的职责边界：内容 vs 文件名
- grep 的 -r（递归目录）vs -i（忽略大小写）区分

**用户常见错误（重要！）：**

| 错误 | 用户写的 | 正确写法 | 原因 |
|------|---------|---------|------|
| 找文件用了grep | `grep "*.py" /root` | `find /root -name "*.py"` | 找文件名用 find |
| find 参数用到grep上 | `grep -name "hello" ...` | `grep "hello" 文件名` | -name 是 find 的参数 |
| 找内容用了find | `find "nologin" /etc/passwd` | `grep "nologin" /etc/passwd` | 在文件内容里找用 grep |
| 目录递归缺 -r | `grep "error" /var/log` | `grep -r "error" /var/log` | 目录下搜要加 -r |

## 补充：find -type 参数（2026-06-14 用户追问）

用户在复习时对 `-type f` 不理解，单独讲解后掌握。

**有效类比：餐厅点菜**
- `-type` = "我要什么类型的菜"（文件 vs 目录）
- `-name` = "我要叫什么名字的菜"

```bash
find /root -type f -name "*.py"    # 只找文件（不找目录）
find /var -type d -name "log*"     # 只找目录
find /usr/bin -type l              # 只找软链接
```

| 参数 | 找什么 | 记忆 |
|------|--------|------|
| `-type f` | 普通文件 | f = file |
| `-type d` | 目录 | d = directory |
| `-type l` | 软链接 | l = link |

**为什么不加 -type 也行？** 不加时 find 文件+目录都出来，加了精确过滤。
用户理解了"为什么需要"后，用法就记住了。
