# 操作前确认规范

## 核心原则

> **听到模糊指令，先停3秒，问清楚再动手。**

用户说任何含有**破坏性**、**执行性**或**修改性**的词汇时，必须追问确认，不能立即执行。

---

## 🔴 必须追问的关键词

### 1. 破坏性词汇（高危）
| 关键词 | 确认什么 |
|--------|----------|
| **删掉**、删除、去掉、清理、清空 | 具体路径？彻底删除还是仅停用？ |
| **关闭**、停用、禁用、卸载、停止 | 影响范围？是关闭功能还是删配置？ |
| **重置**、还原、恢复默认 | 数据备份了吗？范围是全部还是部分？ |

### 2. 执行性词汇（中危）
| 关键词 | 确认什么 |
|--------|----------|
| **开始**、做吧、启动、运行 | 具体任务是什么？有前置条件吗？预算多少？ |
| **执行**、部署、发布、上线 | 影响范围？是测试环境还是生产环境？ |
| **生成**、创建、新建 | 创建在哪里？命名规范？覆盖现有吗？ |

### 3. 修改性词汇（低危但易错）
| 关键词 | 确认什么 |
|--------|----------|
| **改一下**、调一下、换一下 | 改哪里？改成什么样？有参考吗？ |
| **更新**、升级、刷新 | 更新到什么版本？会中断服务吗？ |

---

## 🎯 标准追问话术

### 模板
```
确认一下：你说"[用户原话]"，是指：
- 选项A：[具体解释A]
- 选项B：[具体解释B]
请明确一下，我再执行。
```

### 实际案例

**案例1：误删表情包数据库（2024-06-07 血的教训）**
- 用户说："删掉吧"
- ❌ 错误做法：直接删除 `meme_db/` 目录（包含生产文件）
- ✅ 正确做法：应该问"是指删掉 V2 半成品（502张图在 `meme_cache_v2/`），还是连生产库（`meme_db/` 的5693张索引）一起删掉？"

**案例2：启动服务**
- 用户说："开始吧"
- ❌ 错误做法：立即启动所有服务
- ✅ 正确做法：问"是指开始部署应用到生产环境，还是仅启动本地测试服务？"

## 🛡️ 操作前检查清单

执行任何可能产生副作用的操作前，先问自己：

1. **范围明确吗？** — 知道具体影响哪些文件/服务/用户？
2. **可逆吗？** — 错了能撤回吗？有备份吗？
3. **用户确认了吗？** — 用户明确说"执行"或"确认"了吗？
4. **路径正确吗？** — 特别是删除操作，路径是否包含通配符（`*`）或上级目录（`..`）？

---

## ⚠️ 常见陷阱

- **路径缩写陷阱**：`~` 是家目录，`/` 是根目录，`..` 是上级 — 删除前用 `pwd` 确认当前位置
- **通配符陷阱**：`rm *.py` 可能删错，`rm -rf /` 直接系统毁灭
- **相对路径陷阱**：`cd meme_db` 和 `cd /root/.hermes/meme_db` 是完全不同的
- **用户简称陷阱**：用户说"那个文件夹"，你要问"是指 `meme_cache_v2/` 还是 `meme_db/`？"

## 📋 API 配置专项（铁律）

> **修改 API Key / provider 配置时，必须遵循标准流程，不能直接改 .env。**

### 标准流程（三步走）

| 步骤 | 动作 | 检查点 |
|:---|:---|:---|
| **1. 列参数** | 把完整配置（provider/model/api_key/base_url/default_model）列出来给用户确认 | 用户说"确认"才继续 |
| **2. 写 config** | 直接写入 config.yaml 的 model 段，不用 api_key_env 间接引用 | 验证配置正确 |
| **3. 重启确认** | 问用户"现在重启网关吗？" | 等用户明确说"重启" |

### ❌ 禁止行为

- 直接改 .env 绕过确认流程
- 用 `api_key_env` 间接引用（网关不保证正确读取）
- 不确认就擅自配置
- 不问就自动重启网关

### 正确示例

```python
# 用户说"帮我加个千帆API"
# 第1步：列参数
"配置参数如下，确认后我写入 config.yaml：
- provider: custom
- model: qianfan-code-latest
- base_url: https://qianfan.baidubce.com/v2/coding
- api_key: bce-v3/ALTAKSP-xxx（新Key）
确认吗？"

# 用户说"确认"
# 第2步：写 config.yaml
# 第3步：问用户是否重启
"配置已写入。现在重启网关吗？"
```

## 📋 引用本skill的场景

- 用户说"删掉/去掉/清理"等破坏性词汇
- 用户说"开始/做吧/启动"等执行性词汇但未明确范围
- 用户指令涉及文件系统修改（创建、删除、移动、覆盖）
- 用户指令涉及服务启停、配置更改、数据迁移
