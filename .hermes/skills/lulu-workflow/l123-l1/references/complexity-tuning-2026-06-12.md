# L1 复杂度阈值调优记录（2026-06-12）

## 问题背景

L2 编排器依赖 L1 的 `is_complex` 标志触发。但多步骤任务消息被判 `is_complex=False`，导致 L2 不触发。

## 根因分析

| 问题 | 具体表现 | 根因 |
|------|---------|------|
| `_COMPLEX_MIN_LENGTH=10` | "帮我写脚本并部署"(8字)被拒 | 门槛过高 |
| routing.yaml 缺顿号模式 | "做备份、清缓存、重启" → False | 只认连接词,不认`、` |
| 最小匹配过严 | ".{3,}并.{3,}" → "部署"只有2字不匹配 | 门限应≤中文最短动宾 |
| `、` 不在 _COMPLEX_SEPARATORS | 顿号拆不出子任务 | 缺分隔符 |
| pycache 过期 | 改 routing.yaml 后 pattern 不生效 | 旧 `.pyc` 覆盖 |

## 修复方案

### 1. router.py 修改

```python
# _COMPLEX_MIN_LENGTH: 10 → 6
_COMPLEX_MIN_LENGTH = 6

# _COMPLEX_SEPARATORS: 加顿号
_COMPLEX_SEPARATORS = ["并", "然后", "之后", "接着", "同时", "并且", "再", "随后", "最后", "、"]
```

### 2. routing.yaml 新增模式

```yaml
complex_task_patterns:
  - pattern: ".{2,}、.{2,}"         # 顿号并列
    example: "写爬虫、部署、测试"
  - pattern: ".{2,}之后.{2,}"       # 之后
    example: "做完之后验证"
  - pattern: ".*部署.*部署"          # 多次提及部署
  - pattern: ".*写.*写.*"           # 多个"写"
  - pattern: ".*做.*做.*"           # 多个"做"
```

同时降低已有模式的最小匹配：
- `. {3,}并.{3,}` → `.{2,}并.{2,}`
- `.{5,}再.{5,}` → `.{3,}再.{3,}`
- `.{3,}、.{3,}` → `.{2,}、.{2,}`
- `.{4,}之后.{4,}` → `.{2,}之后.{2,}`

### 3. pycache 清理（关键！）

改 agent/ 下任何 `.py` 文件后**必须清缓存**，否则网关加载旧 `.pyc`：

```bash
find /root/.hermes/hermes-agent/agent/__pycache__ -delete
```

已加入 `/root/run-hermes-gateway.sh` 启动脚本，每次重启自动清理。
