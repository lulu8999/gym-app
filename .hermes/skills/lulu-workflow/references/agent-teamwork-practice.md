# Agent 团队协作实践案例

> 日期：2026-06-09
> 事件：观看抖音视频后创建协作文档

## 背景

用户连续观看两个关于 Hermes Agent 的抖音视频：
1. 《掌握Hermes Agent的7个等级》— 作者：一蛙AI
2. 《HermesAgent 团队模式怎么玩》— 作者：古法编程-小周

视频内容涉及 Hermes Agent 的多 Agent 协作模式，用户要求：

> "看看这个视频，对我们现在的agent协同工作有没有可取之处，你给个plan"

## 我们的现状

| Agent | 部署 | 能力 |
|:------|:-----|:-----|
| Hermes | VPS | 意图理解、任务调度 |
| Claude Code | Mac Mini | 代码生成、系统操作 |
| OpenClaw | Mac Mini | 浏览器自动化 |

## 决策过程

1. **分析视频可取之处**：
   - 团队模式：与我们现有架构高度重合
   - Kanban 多智能体：我们用 delegate_task 做简单并行
   - 意图自动路由：可以减少手动 @

2. **创建 Plan**：
   - L0: 整理现有 Agent 职责文档（立即做）
   - L1: 配置意图自动路由
   - L2: 启用 Kanban Worker 模式
   - L3: 团队监控面板

3. **执行 L0**：创建 `~/.hermes/docs/agent-teamwork.md`

## 结果

用户确认 Plan 后，立即创建了协作文档，包含：
- 团队成员定位表
- 职责边界（谁擅长什么）
- 协作流程图
- 意图路由规则
- 任务交接格式
- 安全边界（案件资料访问控制）
- 故障处理方案
- 待完善功能清单

## 教训

1. **用户说"可以"就立即执行**：用户说"可以"后我没有犹豫，直接创建了文档
2. **文档要写到正确位置**：放到 `~/.hermes/docs/` 而不是 skills 目录
3. **视频内容提取技巧**：当抖音有登录弹窗遮挡时，可以从页面 HTML 描述中提取视频标题和简介
