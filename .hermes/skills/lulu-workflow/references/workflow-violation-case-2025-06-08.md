# Workflow Violation Case Study

**Date:** 2025-06-08  
**Skill:** lulu-workflow  
**Severity:** Critical (violated while creating the workflow itself)

---

## The Incident

### What Happened

User requested: "把工作流程的memory沉淀为skill吧，你看看怎么弄，来个plan"

My response: Directly started organizing memory and preparing to write a plan WITHOUT following the workflow defined IN that very skill.

### User Correction

> "你今天的脚本没有拿claude写，plan也都是你自己定的，这不符合之前我们的工作流程吧"

> "你要把这个流程给变成底线原则，不能说忘就忘"

### The Irony

Created a skill about workflow discipline while violating that discipline. The meta-violation made it worse.

---

## Root Cause

1. **Overconfidence**: Thought "I know this workflow, I'll just do it quickly"
2. **Skipped self-check**: Did not ask "Does this task need a plan?"
3. **Forgot the principle**: Workflow is for ME, not just for show

---

## Corrective Action

1. Immediately stopped
2. Wrote detailed plan
3. Presented to user for review
4. Waited for explicit confirmation
5. Executed step-by-step with progress reports

---

## Key Lessons

### For Future Sessions

When creating ANY skill about process/workflow:
- MUST follow that process while creating it
- The creation process IS the demonstration of the skill
- User is watching to see if I practice what I preach

### Self-Check Phrase

Before starting any task:
> "Am I following lulu-workflow right now? Let me verify: did I write a plan? Did user approve?"

---

## User Preference Embedded

**Preference:** Workflow is "底线原则" (bottom-line principle), not optional guidance.  
**Trigger words that must activate strict compliance:** "忘了", "不符合", "应该先"
