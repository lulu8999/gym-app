# 记忆工具选择指南

## 三层记忆架构

| 工具 | 用途 | 场景 |
|------|------|------|
| **memory tool** | 每次会话自动注入的短期规则 | 用户偏好、临时规则、简单事实 |
| **fact_store (Holographic)** | 长期项目事实 | 架构方案、项目状态、多会话需要记得的东西 |
| **session_search** | 跨会话检索 | 回溯"当时怎么做的" |

## 何时用 fact_store

- 项目架构和状态（像这次的"智能法制助手"项目）
- 长期多阶段任务的关键节点
- 用户说过"不要每次提醒我"的项目
- 需要跨会话保持的复杂上下文

## 何时用 memory tool

- 简单规则（如红线原则）
- 用户偏好（如"测试只发给自己"）
- 临时变量（如当前模型版本）

## 教训

用户说"这个先蒸馏到记忆"时：
- **不要用 memory tool** — 容量有限且每次都要提及
- **用 fact_store** — 外部记忆，永久持有，会话自动注入
