# 非视觉模型下图片消息处理记录（2026-06-05）

## 问题
通过企业微信向圆圆发送问候后，圆圆回复了一张图片。但当前运行的模型是 deepseek-v4-flash（纯文本模型），无法看到图片内容，因此没有回复。用户指出问题："圆圆回你图片你怎么不回复，不能用mimo的视觉模型吗？"

## 根因
- deepseek-v4-flash 是纯文本模型，不支持图片输入
- 系统在收到图片消息时**静默忽略**，没有报错也没有说明
- 用户视角：发了图没回应 → 系统好像坏了

## 教训
1. 非视觉模型收到图片时不能沉默——需要说明情况
2. 如果用户明确要求看图回复，应切换到 mimo-v2.5（支持视觉）
3. 企业微信 wecom_callback 通道默认丢弃非 text/event 类型消息（含图片），这意味着圆圆发的图片可能根本没进 Hermes 会话。但如果图片进了会话，当前模型必须支持 vision 才能处理。

## 企业微信图片消息接收限制
wecom_callback.py 第 348 行只处理 `text` 和 `event` 类型：
```python
if msg_type not in {"text", "event"}:
    return None
```
图片消息（`MsgType=image`）被静默丢弃。如需支持图片需要修改此代码。

## 正确响应模式
- 知道模型不支持视觉 → 主动告知用户
- 用户要求处理图片 → 切换模型
- 发送文件/图片给用户 → 走企业微信 Corp API（`send_wecom.py`）
