## 电池选型

### 接口说明
- **MX1.25** = Molex PicoBlade，间距 1.25mm
- 也叫 Micro JST 1.25、PH 1.25
- 2P（红+/黑-）为主流

### 容量 vs 厚度

| 容量 | 电芯型号 | 厚度 | 续航 |
|------|---------|------|------|
| 500mAh | 502030 | 5mm | ~1.5h |
| 1000mAh | 952540 | 9.5mm | ~3-4h |
| 2000mAh | 603450 | 6mm | ~6-8h |
| 3000mAh | 104050 | 10mm | ~10h |

### ⚠️ 电池+外壳兼容性陷阱
**买电池前必须确认外壳内部空间！** 很多 3D 打印外壳是给裸板设计的，没预留电池空间。
- 简易壳一般留 3-5mm 间隙 → 只能塞 500mAh（5mm 厚）
- 需要电池 → 改外壳后盖加厚 15mm+，或选薄电池
- **永远先确认外壳内部尺寸，再买电池**

## 可行性边界（重要）

ESP32 是**物理世界接口**（传感器采集 + 屏幕显示 + 物理交互），不是独立处理设备，也不是数据管道。

**核心判断标准**：手机/Mac 能做到但需要掏出来/开机吗？如果是，ESP32 的价值就在这里——零操作成本的常驻存在。

详见 `references/esp32-feasibility-analysis.md`。

## 外接模块排针焊接方向（重要！易混淆）

用户反馈：「排针方向向上」和「向下焊接」容易混淆。必须区分两个概念：

### 概念区分
| 术语 | 指的是 | 含义 |
|------|--------|------|
| 模块方向 | 模块本身怎么放 | 传感器面朝上（朝天花板） |
| 排针焊接 | 排针焊在模块哪面 | 焊在背面，针脚朝下 |

### 正确说法（跟店家沟通用）
> 「排针从模块正面插入，背面焊接，针脚朝下」

### 效果图
```
        ↑ 传感器面朝上（朝天花板）
┌──────────────┐
│  [传感器芯片]  │  ← 正面
└──────┬───────┘
       │ 排针（针脚朝下）
       ↓ 杜邦线从下面插
```

### 各模块排针方向
| 模块 | 模块朝向 | 排针焊接 | 原因 |
|------|---------|---------|------|
| BH1750 光照 | 正面朝上 | 背面焊接（针脚朝下） | 感光面朝天花板 |
| BMP280 气压 | 正面朝上 | 背面焊接（针脚朝下） | 朝上感知气压更准 |
| DHT11 温湿度 | 正面朝上 | 背面焊接（针脚朝下） | 传感器面通风 |
| INMP441 麦克风 | 正面朝上 | 背面焊接（针脚朝下） | 麦克风孔朝上收音 |
| VL53L0X 测距 | 水平朝前 | 根据安装方向调整 | 激光头朝目标 |
| EC11 旋钮 | 正面朝上 | 背面焊接（针脚朝下） | 旋钮露壳外 |
| WS2812B 灯环 | 灯珠朝上 | 背面焊接（针脚朝下） | 灯珠朝上发光 |
| 振动马达 | 随便 | 随便 | 放壳里即可 |

## 电源供电说明

ESP32-S3-Touch-LCD-2.8 供电方式：
- **USB-C 插着** → 板子工作 + 自动给电池充电（充满自停）
- **拔掉 USB-C** → 自动切换电池供电，不中断
- **电池 = UPS 备用电源**，日常就 USB-C 一直插着
- 充电管理芯片内置，不需要单独买充电器
- 电池 Molex 1.25 插头插 BAT 口，红正黑负，插反保护板会断电

## OpenSCAD 外壳设计流程

当用户要定制外壳时，按以下步骤：

1. **收集尺寸**：用户发产品图/规格图 → 用视觉模型读取标注尺寸
2. **确认电池**：电池厚度决定外壳是否需要加厚后盖
3. **写 OpenSCAD 代码**：参数化设计（改参数区就能改整个外壳）
4. **关键模块**：
   - 电池卡槽（四壁+底托+拉手槽）
   - 走线槽（电池仓→板子仓）
   - 螺丝柱（对准安装孔间距）
   - 卡扣后盖（不用螺丝，按入即固定）
   - 按钮孔/接口开孔
   - 扩展余量（四周多留 3mm）
5. **输出**：`.scad` 文件 → 用户本地 OpenSCAD 渲染 → 导出 STL → 代打

### 远程导出 STL 流程（2026-06-14 实战）

当 VPS 没有 OpenSCAD 但 Windows 有时，通过 SSH 远程导出：

```bash
# 1. 在 Win 上用 winget 安装 OpenSCAD
sshpass -p '密码' ssh -p 2222 用户@IP 'powershell -Command "winget install OpenSCAD.OpenSCAD --accept-package-agreements --accept-source-agreements"'

# 2. 确认安装路径（winget 不更新 PATH）
# 通常在 C:\Program Files\OpenSCAD\openscad.com

# 3. 传输 .scad 文件到 Windows
sshpass -p '密码' scp -P 2222 model.scad 用户@IP:"C:/Users/用户/Desktop/"

# 4. 分别导出前外壳和后盖
sshpass -p '密码' ssh -p 2222 用户@IP '"C:\Program Files\OpenSCAD\openscad.com" -o "C:\Users\用户\Desktop\front.stl" "C:\Users\用户\Desktop\export_front.scad"'
sshpass -p '密码' ssh -p 2222 用户@IP '"C:\Program Files\OpenSCAD\openscad.com" -o "C:\Users\用户\Desktop\back.stl" "C:\Users\用户\Desktop\export_back.scad"'
```

**分离导出技巧**：写两个包装 `.scad` 文件，用 `use <主文件.scad>` 导入后只调用一个模块：
```scad
// export_front.scad
use <esp32_case.scad>
front_case();
```

**渲染图方案**：VPS 没有 OpenSCAD 时，用 Python matplotlib 画技术图/流程图/截面图（安装流程图、接口布局图等），中文需安装 `pip3 install matplotlib` + 中文字体（WenQuanYi/Noto Sans CJK）。

### 外壳设计原则（用户偏好）
- **安装便利为第一原则**
- 卡槽固定电池（不用双面胶）
- 后盖用卡扣（不用螺丝）
- 留好扩展余量（后续加装传感器/天线等）
- 按钮孔侧面开（针戳即可）

### V6 内部固定结构（重要！用户核心关切）

**设计原则**：外壳不是"纯壳子"，必须有内部固定结构防止模块晃动脱落。

| 编号 | 结构类型 | 适用模块 | 实现方式 |
|------|---------|---------|---------|
| ❶ | 电池仓围栏 | 电池 | 四面挡板（高12mm），底面限位 |
| ❷ | 铜柱安装 | ESP32主板 | M2.5铜柱×4（高18mm），背面螺丝拧入 |
| ❸ | L型卡槽 | DHT22 | 右壁上下卡条+底托，模块滑入卡住 |
| ❹ | L型卡槽 | BME680 | 右壁上下卡条+底托，独立卡槽 |
| ❺ | 托盘式卡槽 | TSL25911+VL53L0X | 前面底部左右挡条+底托 |
| ❻ | L型卡槽 | INMP441 | 左壁上下卡条+底托，拾音孔朝外 |
| ❼ | 铜柱安装 | EC11+OLED模块 | M3铜柱×4（高28mm），架在主板上方 |
| ❽ | 环形卡槽 | WS2812灯环 | 顶面环形凹槽，灯环嵌入卡住 |
| ❾ | L型卡槽 | 振动马达 | 右下角左右挡条+底托 |
| ❿ | 走线槽 | 杜邦线 | 底板主槽+分支槽，线走槽内不乱跑 |

**卡槽类型说明**：
- **L型卡槽** = 上下两条凸起卡条 + 底部托板，模块从侧面滑入卡住
- **托盘式** = 三面围挡（左右+底），模块放入后被限位
- **环形卡槽** = 在顶面开一圈凹槽，灯环嵌入卡住，不用胶水可拆卸
- **铜柱安装** = 底部伸出圆柱，顶部有螺丝孔，模块用螺丝固定

**设计顺序**：先确定所有模块位置 → 设计对应卡槽 → 验证卡槽不互相干涉 → 再画外壳外形

## 外壳设计流程（2026-06-14 总结）

### 3D 可视化技术（matplotlib 3D）

VPS 无 OpenSCAD 时，用 matplotlib 的 `mpl_toolkits.mplot3d` 画精细 3D 图：

**核心函数**：
```python
from mpl_toolkits.mplot3d.art3d import Poly3DCollection

def draw_box(ax, x, y, z, dx, dy, dz, color, alpha=0.6, ec='#333', lw=0.3):
    verts = box_verts(x, y, z, dx, dy, dz)  # 8个顶点
    faces = box_faces(verts)                  # 6个面
    poly = Poly3DCollection(faces, alpha=alpha, facecolor=color, edgecolor=ec, linewidth=lw)
    ax.add_collection3d(poly)

def label_3d(ax, x, y, z, text, color='#333', fontsize=7):
    ax.text(x, y, z, text, fontsize=fontsize, color=color, fontweight='bold', ha='center')
```

**两种视角**：
1. **爆炸图**（elev=25, azim=-55）：各组件沿Z轴分层展开，展示每个独立部件
2. **组装截面**（elev=35, azim=-50）：组件装入实际位置，外壳半透明展示内部

**标注要求**：
- 每个卡槽/安装柱/走线槽都要用 `label_3d()` 标注编号和名称
- 颜色编码：同类型结构用同一色系（如所有L型卡槽用绿色系）
- 底部加图例说明每种结构的功能

**输出**：`fig.savefig(path, dpi=150, bbox_inches='tight', facecolor='white')`

### 设计原则
1. **所有模块一次性考虑** — 设计外壳时必须预留后续外接模块的安装位、走线槽、开口，不能只考虑板子本身
2. **先出方案再动手** — 用户要求先看详细 plan + 安装流程图，确认后再写代码
3. **参数化设计** — 用 OpenSCAD 写参数区，改参数即可调整整个外壳，不改代码逻辑
4. **STL 分件导出** — 前外壳和后盖分别导出 STL，方便单独修改和打印

- 中文字体：**必须显式加载**，代码如下：
  ```python
  font_path = '/usr/share/fonts/chinese/NotoSansCJKsc-Regular.ttf'
  fm.fontManager.addfont(font_path)
  prop = fm.FontProperties(fname=font_path)
  plt.rcParams['font.family'] = prop.get_name()
  ```
- 不要用 `plt.rcParams['font.sans-serif']` 列表方式，会显示方块
- 图片发到微信用 `MEDIA:/path` 标签

### STL 导出流程（通过 SSH 到 Windows）
1. `winget install OpenSCAD.OpenSCAD` — 静默安装
2. `scp` 传 `.scad` 文件到 Windows 桌面
3. 写两个包装文件（`export_front.scad` / `export_back.scad`）用 `use <xxx.scad>` + 调用模块
4. `openscad.com -o output.stl export_xxx.scad` — 命令行导出
5. 注意：Windows 上用 `openscad.com` 而不是 `openscad.exe`（com 版支持命令行）

### 电池供电说明（用户常见疑问）
- USB-C 插板子 → 自动给电池充电 + 板子工作
- 不需要单独充电器，板子内置充电管理芯片
- 电池相当于 UPS 备用电源
- 电池 Molex 1.25 插头插 BAT 口，红正黑负

## Pitfalls

1. **不要相信搜索引擎的价格** — 永远让用户去实际店铺确认
2. **不要跳过电池+外壳兼容性检查** — 塞不进去是最常见的翻车
3. **不要假设所有 2.8寸板子尺寸一样** — 微雪和 CYD（Cheap Yellow Display）不兼容
4. **烧录时要按 BOOT 键** — 忘了这步会以为板子坏了
5. **WS2812B 144灯全亮功耗大** — 需要 5V 3A+ 电源，USB 口供电可能不够
6. **激光模组要注意安全** — 532nm 绿光虽然只有 5mW，但不能直射眼睛
7. **图片识别要互补** — vision_analyze 识别不全时，用 MiMo 视觉模型补充，两者互为备选
8. **电池先确认外壳再买** — 外壳没设计好就买电池，大概率塞不进去或浪费钱
9. **产品规格图 > 搜索引擎** — 收到用户发的规格图时，以图上标注尺寸为准，不要靠猜
12. **用户问"插手机提取数据"时** — ESP32 USB 是 Device 端不是 Host 端，手机也是 Device，两个对着插不行。而且微信数据 SQLCipher 加密，密钥在安全芯片，ESP32 算力 1/1000 不够破解。直接说明局限，给出替代方案（手机 App + VPS 架构），见 `references/esp32-feasibility-analysis.md`。
13. **OpenSCAD 在 RHEL/OpenCloudOS 上不可用** — `dnf install openscad` 无包。用 Python matplotlib 渲染 2D 技术图+安装流程图作为替代方案。代码见 `/root/esp32-shell/render_images.py`，输出在 `/root/esp32-shell/output/`。如果需要真 3D 渲染，让用户在本地装 OpenSCAD（Windows/Mac 都有安装包）。**中文字体关键**：用 `fm.fontManager.addfont('/usr/share/fonts/chinese/NotoSansCJKsc-Regular.ttf')` 显式加载，`plt.rcParams['font.sans-serif']` 列表方式不可靠。
10. **用户问"插手机提取数据"时** — ESP32 USB 是 Device 端不是 Host 端，手机也是 Device，两个对着插不行。而且微信数据 SQLCipher 加密，密钥在安全芯片，ESP32 算力 1/1000 不够破解。直接说明局限，给出替代方案（手机 App + VPS 架构），见 `references/esp32-feasibility-analysis.md`。
11. **不要把 ESP32 定位为"数据中转桥"** — 手机可以直接 WiFi/AirDrop 到 Mac，加 ESP32 中转多此一举。用户会问"为什么不直接传"。ESP32 的价值是"永远在桌上的物理接口"，不是"数据管道"。
- 11. **用户问"树莓派是不是更好"时** — 树莓派定位和 Mac 重叠（都是通用计算机），无自带屏幕/电池管理，开机慢，体积大。ESP32 是"智能家电"，树莓派是"小电脑"，不同赛道。见 `references/esp32-feasibility-analysis.md`。
- 12. **ESP32 可以做 USB 转网络的数据桥梁** — 用户提出：ESP32 Type-C 接口连目标设备（U盘、手机等），通过 WiFi 把数据传给 Mac 处理，结果返回 ESP32。ESP32-S3 支持 USB OTG（GPIO19/20），可当 USB Host 连接外部设备。这是"ESP32 当前端搬运工、Mac 当大脑"的架构，和之前的"数据中转桥"概念不同——这次是 ESP32 通过 USB 主动连接其他设备，而不是被动被连。
- 13. **法治助手设备架构（2026-06-14 讨论）** — VPS 做大脑（服务/API）、Mac 做仓库（PostgreSQL/备份）、ESP32 做 USB 桥梁（连接外部设备）、终端设备做界面（浏览器访问）。VPS 通过反代（Nginx reverse proxy）转发 Mac 上的加密数据服务，普通功能直接跑 VPS。
13. **排针焊接方向要明确** — 用户问"向下焊接就是排针朝上吗"，说明术语不直观。跟用户解释时画图：模块正面朝上，排针从上面穿下去焊在背面，杜邦线从下面引出。跟店家说「排针焊好，针脚朝下」。
14. **ESP32 电池供电原理** — 用户问"怎么给电池供电"，答案就是插 USB-C。板子内置充电芯片，USB-C 插着就充电+工作，拔了自动切电池。跟手机一样，不需要单独充电器。
15. **DHT22 买便宜的就行** — DHT22 有 ¥5.4 的裸传感器版和 ¥24.66 的蓝色 PCB 板版，参数完全一样，贵的只是多焊了一块小板。让用户买便宜的。
16. **外壳设计分两步** — 先设计板子+电池的基础外壳，等用户拿到外接模块实物后再加模块固定位。不要在没有实物尺寸的情况下瞎猜模块布局。
17. **Windows 装 OpenSCAD 用 winget** — `winget install OpenSCAD.OpenSCAD` 能装上，但 PATH 不会立即更新，SSH 会话里 `openscad` 命令找不到。用完整路径 `C:\Program Files\OpenSCAD\openscad.com`。导出 STL：`openscad.com -o output.stl input.scad`。
18. **VPS 无 OpenSCAD 时用 matplotlib 替代** — 画技术图纸（正视图/侧视图/爆炸图/接口图/安装流程图/截面图），中文显示需装 `pip3 install matplotlib`，字体用 `Noto Sans CJK SC`。**关键**：必须用 `fm.fontManager.addfont(字体路径)` + `plt.rcParams['font.family'] = prop.get_name()` 显式加载字体，光靠 `plt.rcParams['font.sans-serif']` 列表不够（会显示方块）。VPS 字体路径：`/usr/share/fonts/chinese/NotoSansCJKsc-Regular.ttf`。
19. **用户问"针脚拿来干啥"时** — 用插座比喻：针脚=插座，模块=电器，杜邦线=电线。简单类比比列引脚定义有效得多。
13. **外壳材料选择** — 桌面设备 PLA 够用；移动/执法场景考虑铝合金 CNC（耐摔、散热好）。折中：先 PLA 验证尺寸再 CNC。
14. **花盆湿度监测** — 米家花花草草 ¥90/个太贵，电容式有线传感器 ¥3-5/个性价比最高。详见 `references/soil-moisture-sensor-research.md`。
15. **同型号不同版本价格差很大** — DHT22 有 ¥5.4 裸传感器版和 ¥24.66 蓝色 PCB 板版，参数完全一样，贵的只是多焊了一块小板。看到价格差异大的同型号产品，先对比参数是否一致，再决定买哪个。
16. **桌面终端屏幕朝上放，不要朝前** — 用户明确说"为什么不把屏幕仰面朝上"。桌面终端应该像控制台一样平放，屏幕朝天花板，低头就能看。正面朝前的布局需要支架，不如朝上自然。旋钮放右侧（右手操作），OLED 小屏放顶面另一侧做信息面板。
17. **EC11 的小 OLED 要利用起来，放屏幕上方当状态栏** — EC11+OLED 集成模块自带 1.3寸 OLED（128×64），在外壳顶面触摸屏正上方开窗口外露，常显状态信息（时间/消息/电量/天气）。用户明确说"OLED放在触摸屏上面"，不要放旋钮下方（会溢出右边界）。
16. **模块尺寸以实物图标注为准，不要凭记忆估** — INMP441 最初估为 10×10mm，用户发实物图后确认是 φ14mm 圆形。外壳设计前必须用 vision_analyze 读取产品图上的标注尺寸，不能靠搜索引擎或记忆估算。
17. **用户说"我一个一个发，你确认了跟我说，要记住了"时** — 逐个确认模式：每收到一张图→vision_analyze 读尺寸→表格确认→"已记下，下一个 👍"→全部确认完后更新 skill。不要催、不要跳、不要一次性问所有图。
16. **模块"背面"判断** — 用户发产品图问"这是正面还是背面"时，看传感器芯片在哪面：芯片朝外的是正面（测数据用），焊盘/排针通常在背面。但有些模块（如 BME680）焊盘和传感器在同一面，这种排针焊正面就行，传感器朝上、排针朝下不矛盾。
17. **hermescli + ESP32 触控屏的瓶颈是键盘输入** — 2.8 寸触摸屏打字太痛苦，解决方案：蓝牙折叠键盘（¥30-50）或屏幕快捷短语按钮（一键调取常用指令）。详见 `references/sensor-shopping-list.md` 法制助手额外建议。
18. **传感器升级决策框架** — 用户问"有没有更强的"时，按场景推荐：弱光环境→TSL2591（光照），需要空气检测→BME680（气压+VOC），测距翻倍→VL53L1X。每次升级多花 ¥5-20，但功能有质的提升。详见 `references/sensor-shopping-list.md` 升级方案。
13. **winget 安装后 PATH 不更新** — winget 装的软件路径不在当前 SSH session 的 PATH 中，必须用完整路径调用（如 `C:\Program Files\OpenSCAD\openscad.com`）。用 `Get-ChildItem "C:\Program Files\XXX"` 或 `Get-ChildItem "C:\Program Files (x86)\XXX"` 找到实际安装位置。
14. **先出设计方案再买材料** — 外壳设计决定电池尺寸，电池尺寸决定外壳空间。先画好 OpenSCAD 模型确认尺寸无误，再让用户买电池和找代打，避免返工浪费。
15. **铝合金 vs PLA 材质选择** — 执法/户外场景建议铝合金 CNC 加工（耐摔、散热好、质感专业），桌面/原型阶段用 PLA 3D 打印。先打 PLA 验证尺寸，确认后再做铝合金版。
16. **效果图不要画参考线/虚线圆圈** — 用户看到外壳效果图里的虚线参考线（如灯环外径/内径边界线）会困惑"那是什么"。效果图只画实际可见的组件（LED灯点、屏幕、旋钮），尺寸标注用箭头和文字，不要画额外的几何参考线。
17. **每个组件必须验证不溢出外壳边界** — 设计时逐一计算：组件中心±半径 ≤ 外壳半宽/半高。溢出的组件（如大灯环）要么缩小、要么外壳加宽、要么嵌入壁厚（需向用户说明）。
18. **WS2812 灯环选型要考虑外壳适配** — 16灯版外径67.8mm在108mm壳里会溢出（需嵌入壁厚13.9mm），12灯版外径49.9mm完全包在壳内。设计前先确认灯环外径 vs 外壳宽度，别选了灯环才发现塞不下。
19. **EC11+OLED 模块很大（65.3×36.1mm），外壳要足够宽** — 这个模块比想象中大，外壳宽度至少要 100mm+ 才能放下。灯环绕旋钮时总宽度 = 旋钮中心 + 灯环半径，必须验证右边缘不溢出。
20. **OLED 状态栏放屏幕上方，不放旋钮下方** — 用户明确要求"OLED放在触摸屏上面"。旋钮下方放 OLED 会导致 OLED 超出外壳右边缘（旋钮中心38+15=53>50mm）。放屏幕上方更紧凑、更直觉。
21. **外壳必须有内部固定结构，不能是"纯壳子"** — 用户明确批评："就是一个纯壳子，里面没有设计能卡住的卡槽吗，这样晃一晃不全都没了"。设计外壳时必须为每个模块设计对应的卡槽/安装柱/托盘，模块要能卡住不晃。详见 V6 内部固定结构表。
22. **用户要看精细的3D立体图带标注** — 用户说"把这个三维立体图做的精细一点，把卡槽什么的都标出来"。2D平面图不够直观，必须用 matplotlib 3D 画爆炸图+组装截面图，每个卡槽/安装柱/走线槽都要标注编号和名称。
23. **OLED与屏幕之间不要实体隔断条** — 用户说"不用实体隔断条，直接把两个屏幕的距离稍稍拉远一点就行"。用5mm间距自然分隔，比加隔断条更简洁美观。
24. **效果图不要画参考线/虚线圆圈** — 用户看到外壳效果图里的虚线参考线（如灯环外径/内径边界线）会困惑"那是什么"。效果图只画实际可见的组件（LED灯点、屏幕、旋钮），尺寸标注用箭头和文字，不要画额外的几何参考线。
25. **有屏幕的设备，3D图必须验证组件是否遮挡屏幕** — V6的3D爆炸图把EC11放在ESP32主板正上方，会挡住屏幕，用户一眼就发现了。教训：2D顶视图看不出来（因为是俯视），3D图才能暴露Z轴堆叠问题。设计有屏幕的外壳时，必须用3D视角检查：其他组件是否在屏幕正上方/正前方遮挡显示区域。宁可外壳加宽（V7从100→112mm让EC11水平并排），也不要堆叠挡住屏幕。
26. **用户会检查3D图并发现物理问题** — 不要假设用户只看2D图。用户会仔细审视3D爆炸图和截面图，发现堆叠遮挡、走线冲突等物理问题。所以3D图必须准确反映实际安装位置，不能随便摆放组件。
27. **灯环尺寸必须验证边缘到外壳壁的余量** — 灯环中心位置确定后，必须计算：灯环右边缘 = 中心X + 外径/2，左边缘 = 中心X - 外径/2。右边缘必须 ≤ 外壳右壁(W - wall)。本项目两次犯错：第一次EC11右移到x=76后灯环中心在x=91，右边缘116mm > 外壳110mm壁；第二次同样的错误。最终方案：外壳加宽到120mm，灯环余量=120-2-116=2mm。**设计灯环位置时先算边缘再定中心。**
28. **EC11模块（65.3mm宽）+主板（69mm宽）=134.3mm > 100mm外壳宽度，必须水平并排且外壳加宽** — 两个模块加起来比100mm外壳宽得多，不能堆叠（会挡住屏幕），只能水平并排。最终外壳120mm，主板x=3~72，EC11 x=76~106，中间4mm间隙。
29. **matplotlib 3D 渲染代码在 execute_code 中容易被用户消息中断** — 3D渲染（Poly3DCollection + 多个draw_box）耗时较长，execute_code 在等待结果时用户发消息会中断执行。**解决方案**：先用 write_file 写脚本文件，再用 terminal(background=true, notify_on_complete=true) 后台运行，最后 process(action=poll) 检查结果。不要在 execute_code 里跑复杂3D渲染。
30. **V7最终布局坐标** — 外壳120×85×36mm，壁厚2mm。ESP32主板：x=3~72, y=17.55~67.45。EC11模块：x=76~106, y=24.45~60.55。灯环中心：x=91, y=42.5，外径49.9mm，右边缘115.95mm，距右壁(118mm)余量2.05mm。安装柱：ESP32用M2.5×18mm，EC11用M3×22mm。
31. **复杂3D渲染失败时用2D替代** — matplotlib Poly3DCollection 渲染10+个组件时，execute_code 频繁被用户消息中断（代码太重）。**降级方案**：用 matplotlib patches（FancyBboxPatch + Circle）画2D俯视图，秒出、稳定、信息量够用。2D俯视图能清楚展示模块位置、间距、溢出检测，适合确认布局。3D效果图留到布局确认后、最终出图时再画。
