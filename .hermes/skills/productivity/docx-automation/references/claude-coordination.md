# 与 Claude Code 协作工作流

复杂文档生成任务的分工协作方案。

## 分工原则

| 角色 | 职责 | 资源限制 |
|------|------|---------|
| **Claude Code** (子代理) | 编程/开发/代码审查 | 单文件/591MB内存/最多2并行 |
| **我** (主代理) | 协调/验证/汇报/系统操作 | 全局管理 |

## 工作流程

```
1. 我接收任务
2. 我分析需求，制定 Markdown 结构方案
3. Claude Code 按方案生成完整 Markdown 内容
4. Claude Code 编写/优化转换代码（如需要）
5. 我调用 md_to_office 工具进行转换
6. 我验证输出质量并汇报
```

## 实例：修改306考研资料

**步骤分解**：

1. **我分析现有文档**
   - 使用 `docx_utils.analyze_docx_structure()` 分析
   - 发现表格内容被拆散成 575 个段落
   - 确定需要重建表格结构

2. **我制定修复方案**
   - 确定章节结构
   - 规定 Markdown 格式约定
   - 列出需要生成的表格

3. **Claude 生成 Markdown**
   - 按方案生成完整的 Markdown 内容
   - 包含标题、列表、表格、口诀

4. **Claude 写转换工具**
   - 如需要定制转换逻辑
   - 生成辅助脚本

5. **我执行转换**
   - 使用 `md_to_office.convert_md()` 转换
   - 验证输出质量

6. **我汇报结果**
   - 检查段落数是否合理
   - 检查表格是否完整
   - 发送结果文件给用户

## 什么时候用这个模式

**用这个模式**：
- 任务需要写代码（生成工具脚本）
- 内容很长很复杂（需要结构化思维）
- 需要代码审查或调试

**不用这个模式**（我自己干）：
- 简单的格式调整
- 已有工具可用（直接调用）
- 系统操作（文件管理、命令执行）

## 常见坑与解决

### Claude 生成的 Markdown 不符合预期
**解决**：提供更详细的格式示例

### 转换后格式乱了
**解决**：
1. 检查 Markdown 语法是否标准
2. 确保表格有分隔行 `|---|---|`
3. 确保标题层级正确

### 任务执行时间太长
**解决**：
- 考虑使用后台任务模式（见 task-manager skill）
- 将大任务拆分为多个小任务

python3 -c "

    toolsets=['terminal', 'file']
