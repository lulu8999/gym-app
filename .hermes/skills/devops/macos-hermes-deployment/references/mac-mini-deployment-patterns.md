# Mac Mini M4 部署模式参考

本文件记录 Lulu 的 Mac Mini M4 实际部署规划，作为类似场景的参考模板。

## 硬件规格

- **设备**: Mac Mini M4 (16GB RAM + 256GB SSD)
- **扩展计划**: +1TB SSD (数据) + 4TB 机械硬盘 (敏感数据)
- **网络**: 无公网 IPv4，使用 Tailscale + Cloudflare Tunnel

## 存储分层规划

| 存储 | 容量 | 用途 | 数据类型 |
|------|------|------|----------|
| 系统 SSD | 256GB | macOS + Hermes 程序 + 配置 | 可快速重建 |
| 数据 SSD | 1TB (后续加) | Hermes 数据、数据库、日志 | 需要备份 |
| 机械硬盘 | 4TB (后续加) | 智能法制办案助手 + 案件资料 | **敏感数据，本地优先** |

## 迁移策略 (10天准备期)

```
┌─────────────────────────────────────────────────────────┐
│  现在（准备期，10天）                                      │
├─────────────────────────────────────────────────────────┤
│  VPS (主力)          Mac Mini (准备中)                    │
│  ├─ Hermes 全功能    ├─ Hermes 基础版（纯净）              │
│  ├─ 表情包/娱乐      ├─ Claude Code                      │

│  └─ 企微资料         └─ 不处理敏感数据                    │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│  10天后（切换完成）                                        │
├─────────────────────────────────────────────────────────┤
│  VPS (轻量服务)        Mac Mini (主力服务器)               │
│  ├─ 论文处理           ├─ Hermes 全功能                    │
│  ├─ 企微资料存档       ├─ Claude Code                     │
│  └─ 辅助服务           ├─ 智能法制办案助手                  │
│                        └─ 案件资料（本地敏感数据）          │
└─────────────────────────────────────────────────────────┘
```

### 数据分离原则

| 数据类型 | 存放位置 | 原因 |
|----------|----------|------|
| 论文处理 | VPS | 已稳定运行，无需迁移 |
| 企业微信资料 | VPS | 归档性质，不常访问 |
| 案件资料 | Mac Mini 本地 | **敏感数据，不出境** |
| 智能法制助手 | Mac Mini 本地 | 涉及专业领域，本地部署 |

## Cloudflare DNS 清理流程

实际清理 `lulugame.fun` 时的验证方法：

| 域名 | 验证方式 | 结果 | 操作 |
|------|----------|------|------|
| `admin.lulugame.fun` | 浏览器访问 | 用户管理面板 | 保留 |
| `callback.lulugame.fun` | 已知用途 | 企微回调 | 保留 |
| `wecom.lulugame.fun` | 已知用途 | 企业微信 | 保留 |
| `chat.lulugame.fun` | 浏览器访问 | 跳转其他网址 | 删除 |
| `hub.lulugame.fun` | 浏览器访问 | 跳转其他网址 | 删除 |
| `term.lulugame.fun` | 浏览器访问 | 无法访问 | 删除 |
| `loc.lulugame.fun` | 用户确认 | 定位服务已废弃 | 删除 |
| `github.lulugame.fun` | 检查记录 | 指向测试IP 192.0.2.1 | 删除 |

### 清理 checklist

- [ ] 列出所有 DNS 记录
- [ ] 逐个访问验证实际用途
- [ ] 标记"跳转/无法访问/测试地址"类记录
- [ ] 与用户确认删除列表
- [ ] 执行删除
- [ ] 保留核心记录（admin, callback, wecom, 根域名）

## 多 Agent 协同架构

```
┌────────────────────────────────────────┐
│           Mac Mini M4                  │
├────────────────────────────────────────┤
│  ┌──────────────┐  ┌──────────────┐   │
│  │   Hermes     │  │ Claude Code  │   │
│  │  (主控制器)   │  │  (复杂编程)   │   │
│  └──────┬───────┘  └──────────────┘   │
│         │                              │
│  ┌──────┴───────┐  ┌──────────────┐   │
│  │  智能法制助手  │  │  OpenClaw    │   │
│  │  (案件分析)   │  │ (浏览器自动化) │   │
│  └──────────────┘  └──────────────┘   │
└────────────────────────────────────────┘
```

### Agent 分工

| Agent | 职责 | 触发条件 |
|-------|------|----------|
| **Hermes (主控)** | 日常对话、任务调度、记忆管理 | 默认入口 |
| **Claude Code** | 复杂编程、代码审查、重构 | 编程任务 > 50行代码 |
| **OpenClaw** | 浏览器自动化、网页操作 | 需要访问网页/截图 |
| **法制助手** | 案件分析、法规查询、文书生成 | 案件相关查询 |

## 用户协作偏好 (Lulu)

### 报告风格
- **走一步报告一步** - 每阶段主动报告状态
- 格式：✅ 已完成 / 🔄 正在执行 / ⏳ 等待中

### 规划优先
- 先讨论架构、存储、域名
- 确认后再执行
- 不急于跑命令

### 敏感数据处理
- 密码让用户自己输入
- 或临时改简单密码，配完改回
- 不在聊天记录留长期凭据
