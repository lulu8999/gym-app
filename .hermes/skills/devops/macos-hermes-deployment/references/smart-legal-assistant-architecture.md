# 智能法制助手 — 架构设计方案

> 📅 方案日期：2026-06-09  
> 👤 设计者：Lulu + 师傅  
> 🎯 目标：案件资料管理 + 文书生成 + 类案检索，降低基层工作压力

---

## 核心需求

| 需求 | 技术对应 |
|------|----------|
| 中文文档（PDF/Word/Excel） | 全文检索 + 内容提取 |
| 20人峰值使用 | PostgreSQL 并发能力 |
| 权限分级 | 角色-用户-案件三级权限 |
| 降低基层工作压力 | 智能助手 + 文书生成 + 快速检索 |

---

## 最终架构（生产级）

```
┌─────────────────────────────────────────────────────────────────┐
│                     Mac Mini M4 (内网核心)                       │
│  ├─ PostgreSQL 16 (案件数据库)                                   │
│  │   ├── 中文全文检索 (pg_jieba 分词)                            │
│  │   ├── 用户权限表 (角色：管理员/侦查员/文职)                    │
│  │   ├── 案件主表 (元数据：案号、类型、日期、状态)                 │
│  │   ├── 文档内容表 (PDF/Word/Excel 提取的文本)                  │
│  │   └── 审计日志 (谁在什么时间查了什么)                          │
│  │                                                              │
│  ├─ 文档处理引擎 (Apache Tika + LibreOffice)                     │
│  │   └── 自动提取上传文件的文本内容                              │
│  │                                                              │
│  ├─ 智能法制助手                                                 │
│  │   ├── 文书生成 (根据案情模板生成文书)                         │
│  │   ├── 案件查询 (自然语言搜索)                                 │
│  │   ├── 串并案分析 (相似案件推荐)                               │
│  │   └── 法规检索 (法条关联)                                     │
│  │                                                              │
│  ├─ Hermes Gateway (微信入口)                                    │
│  │   └── 你发指令 → 我在Mac上执行 → 返回结果                     │
│  │                                                              │
│  └─ Tailscale (20人安全接入)                                    │
│      └── 每个用户分配内网IP，加密访问                             │
└─────────────────────────────────────────────────────────────────┘

VPS (公网轻量入口)
├── 降级模式 (Mac离线时基础应答)
└── 论文/企微备份 (历史数据)

ESP32 (随身移动前端) — 详见三层架构文档
├── USB-C 连接目标设备 (手机/U盘/IoT)
├── WiFi 连接 Mac/VPS
├── 传感器采集 (温湿度/光照/气压)
├── 触摸屏本地交互
└── 纯数据搬运，不做计算
```

## 权限分级设计

| 角色 | 权限范围 | 典型用户 |
|:----:|----------|----------|
| **管理员** | 所有案件 + 用户管理 + 系统配置 | Lulu、师傅 |
| **侦查员** | 自己经手的案件 + 协查案件 | 办案民警 |
| **文职** | 文书录入 + 查询自己录入的案件 | 内勤、辅警 |
| **只读** | 查看指定案件，不能修改 | 领导、督查 |

## 功能模块详情

### 1. 法条匹配

| 数据来源 | 是否需要 OpenClaw | 实现方式 |
|:--------:|:-----------------:|----------|
| 本地法律库 | ❌ 不需要 | PostgreSQL 全文检索（pg_jieba 分词） |
| 本地历史案件 | ❌ 不需要 | SQL 查询 + 向量相似度计算 |
| 网上相似案件 | ✅ **需要** | OpenClaw 浏览器访问 + 抓取 |

**本地法律库建议收录：**
- 刑法 + 修正案
- 刑事诉讼法
- 治安管理处罚法
- 公安机关办理刑事案件程序规定
- 常见罪名司法解释（盗窃、诈骗、故意伤害等）
- 本地量刑指导意见

### 2. 文书生成

- 根据案情模板生成文书
- 支持常见：起诉意见书、调查报告、讯问笔录

### 3. 案件查询

- 自然语言搜索
- 向量相似度计算

### 4. 串并案分析

- 相似案件推荐
- 关联分析

## 存储容量规划

| 存储层 | 容量 | 用途 |
|--------|------|------|
| 内置 SSD | 256G | 系统 + Hermes + 热数据 |
| 外接 SSD | 1TB | 项目代码 + 容器镜像 |
| 机械盘 | 4TB | 案件资料 + 法律库 + 知识库 |

**案件资料预估：**
- 50GB 以内（SQLite 足够）
- 超出后可迁移到 PostgreSQL

---

## 数据策略

| 数据类型 | 存储位置 | 说明 |
|----------|----------|------|
| 案件资料 | Mac 本地加密 | VPS 不留 |
| 论文资料 | VPS | 历史数据 |
| 企微备份 | VPS | 历史数据 |
| 法律条文 | Mac 本地 | 静态数据，适合本地存储 |

---

## 关键决策

1. **Mac 主 + VPS 灾备**：Mac 离线时 VPS 降级模式
2. **案件资料不上 VPS**：仅 Mac 本地加密存储
3. **OpenClaw 常驻**：用于网上相似案件查询
4. **4TB 机械盘**：法制助手专用
