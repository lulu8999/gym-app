# 三层设备架构 — VPS + Mac + ESP32

> 📅 设计日期：2026-06-14
> 🎯 核心理念：VPS 做大脑，Mac 做仓库，ESP32 做手脚

---

## 架构总览

```
用户（微信/企微）
    ↓ 消息
┌─────────────────────────────┐
│  VPS（公网大脑）              │
│  ├── Hermes Gateway（入口）   │
│  ├── Nginx 反代              │
│  ├── API 网关                │
│  └── 普通功能后端            │
└──────────┬──────────────────┘
           │ Tailscale / SSH 隧道
           ↓
┌─────────────────────────────┐
│  Mac（家庭云服务器）          │
│  ├── PostgreSQL（案件数据库） │
│  ├── 加密文件存储            │
│  ├── 数据处理引擎            │
│  ├── 智能法制助手后端        │
│  └── 大模型推理（可选）      │
└──────────┬──────────────────┘
           │ WiFi / 互联网
           ↓
┌─────────────────────────────┐
│  ESP32（随身移动前端）        │
│  ├── USB-C 连接目标设备      │
│  ├── WiFi 连 Mac/VPS        │
│  ├── 数据采集与转发          │
│  └── 传感器/外设控制         │
└──────────┬──────────────────┘
           │ USB-C / 串口 / I2C
           ↓
      目标设备（手机/U盘/IoT设备）
```

## 各层职责

### VPS — 公网入口 + 轻量服务

| 职责 | 说明 |
|------|------|
| Hermes Gateway | 接收微信/企微消息，分发指令 |
| Nginx 反代 | 统一域名入口，转发到 Mac 内部服务 |
| 普通功能 | 法条查询、文书模板、类案检索（非敏感数据） |
| 安全网关 | 身份认证、访问控制、日志审计 |
| 降级模式 | Mac 离线时提供基础应答 |

**域名规划**：
- `lulugame.fun` — 主入口
- `lulugame.fun/api/*` — 普通功能 API
- `lulugame.fun/case/*` — 法制助手（反代到 Mac）

### Mac — 家庭云服务器（固定不动）

| 职责 | 说明 |
|------|------|
| PostgreSQL | 案件数据库，加密存储 |
| 文件存储 | 案件资料、文书、证据文件 |
| 数据处理 | 文档解析、全文检索、向量计算 |
| 智能助手 | 文书生成、串并案分析、法条匹配 |
| 大模型推理 | 本地小模型（可选，Mac Mini M4 16G+） |
| ESP32 管理 | 接收 ESP32 数据，下发指令 |

**关键**：Mac 不暴露公网，所有访问通过 VPS 反代或 Tailscale 内网。

### ESP32 — 随身移动前端（走到哪带到哪）

| 职责 | 说明 |
|------|------|
| USB Host | 连接目标设备（手机、U盘、IoT设备） |
| 数据采集 | 读取目标设备原始数据 |
| 数据转发 | 通过 WiFi 把数据发给 Mac |
| 指令执行 | 接收 Mac 下发的命令，对目标设备执行 |
| 传感器控制 | 温湿度、光照、麦克风等外设 |
| 现场显示 | 触摸屏显示状态/结果 |

**ESP32 不做计算**，只当数据搬运工。所有处理都在 Mac 上完成。

## 连接方式

### VPS ↔ Mac

| 方式 | 用途 | 安全性 |
|------|------|--------|
| Tailscale | SSH 管理、内部 API 调用 | ✅ 私有网络 |
| SSH 隧道 | 端口转发（备用） | ✅ 加密 |
| Cloudflare Tunnel | 公网暴露 Hermes/反代 | ✅ CF 保护 |

### Mac ↔ ESP32

| 方式 | 用途 | 延迟 |
|------|------|------|
| WiFi（同一局域网） | 局域网直接通信 | <10ms |
| WiFi → 互联网 → Tailscale | 外出时远程连接 | 50-200ms |
| WiFi → 互联网 → VPS 中转 | 无 Tailscale 时 | 100-300ms |

### ESP32 ↔ 目标设备

| 接口 | 用途 | 协议 |
|------|------|------|
| USB-C (OTG) | 连手机、U盘 | ADB / Mass Storage / CDC |
| UART (TX/RX) | 连嵌入式设备、路由器 | 串口 115200bps |
| I2C (SDA/SCL) | 连传感器模块 | I2C 协议 |
| SPI | 连高速外设 | SPI 协议 |
| GPIO | 连简单外设（LED、按钮） | 高低电平 |

## 数据流示例

### 示例 1：远程读取手机文件

```
1. 用户微信发命令："读取连接手机的文件列表"
2. Hermes → 转发给 Mac
3. Mac → 通过 WiFi 发 USB 命令给 ESP32
4. ESP32 → USB-C 连手机，执行 ADB shell ls /sdcard/
5. 手机返回文件列表 → ESP32 → WiFi → Mac
6. Mac 格式化结果 → Hermes → 微信推送给用户
```

### 示例 2：环境数据采集

```
1. ESP32 传感器定时采集（温湿度/光照/气压）
2. 数据通过 WiFi → Mac
3. Mac 存入数据库，异常时触发告警
4. 告警通过 Hermes → 企微推送给用户
```

### 示例 3：法制助手现场使用

```
1. 用户在 ESP32 触摸屏上输入案件关键词
2. ESP32 → WiFi → Mac（PostgreSQL 全文检索）
3. Mac 返回匹配案件列表 → ESP32 屏幕显示
4. 用户点击查看详情 → Mac 返回详细信息
```

## 安全设计

### 反代安全层（VPS）

- Nginx 反代 + SSL/TLS
- 身份认证（JWT / API Key）
- 访问频率限制
- 日志审计

### 数据加密（Mac）

- PostgreSQL 透明加密（TDE）
- 文件系统加密（FileVault）
- 传输加密（TLS / SSH）

### ESP32 安全

- WiFi WPA3 加密
- 通信走 HTTPS / MQTT over TLS
- 固件签名验证（防篡改）
- 设备认证（Mac 只接受已注册的 ESP32）
