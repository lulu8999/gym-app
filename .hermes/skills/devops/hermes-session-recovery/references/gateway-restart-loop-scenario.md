# 网关重启循环场景 — 实战案例

## 发生时间
2026-06-04，企微回调配置对话中。

## 现象
网关频繁重启（restarts 达 118 次），每次重启后 agent 重新问同一个问题：
> "域名用 callback.lulugame.fun 行不行？"

用户在第 4 次看到同样的问题后爆发：
> **"你一直重启干啥"**

## 根因分析

1. **网关重启原因** — 未知（可能是 PM2 或网关自身 bug），每次重启产生「gateway shutdown/interrupted」提示
2. **Agent 行为缺陷** — 每次看到中断提示，没有查 PM2 状态就默认对话被重置，从上一个未确认的问题重新开始
3. **没有意识到重复提问** — 第 2 次、第 3 次看到同样的提示时，没有检查历史中是否已经问过同样的问题

## 教训

### 1. 看到中断提示后的正确反应

不要直接问用户——先查状态：
```bash
pm2 status | grep hermes-gateway
# 看 uptime 列：如果 uptime 很短（几秒到几分钟），证明刚刚重启过
```

### 2. 检查历史中的待确认问题

在看历史对话时，如果发现已经有未确认的提问（agent 问了但用户没回复）：
- **不要重复问** — 可能用户已经看到了只是还没回
- **换个方式问** — 比如如果上次问的是「域名行不行」，这次可以问「域名还没确认？需要我换个吗」
- 或者给出默认值自己做掉，不需要每次都问

### 3. 第 1 次重复可能是意外，第 2 次就是 bug

如果发现自己要问第 2 次同样的问题，停下来想想：是不是网关又重启了？是不是刚问过？

## 正确做法

```
[中断提示出现]
→ 检查 pm2 status（网关已重启）
→ 检查配置是否还在（grep config）
→ 看历史里自己最后问了什么
→ 如果问过但用户没回，用不同措辞再问一次，或直接推进
→ 继续任务
```
