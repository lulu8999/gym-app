## 二、核心功能设计

### 1. 总览统计（顶部卡片）
| 指标 | 说明 | 数据源 |
|------|------|--------|
| 总训练次数 | 所有训练记录数 | trainings |
| 总训练时长 | 累计训练时间 | trainings.duration |
| 总训练量 | 累计重量（吨） | trainings.total_volume |
| 连续训练天数 | 打卡统计 | 计算值 |
| 平均每次时长 | 总时长/次数 | 计算值 |

### 2. 训练日历（热力图）🔥
- 类似 GitHub 贡献日历
- 显示最近 90 天训练情况
- 颜色深浅表示训练量
- 点击某天显示当日训练详情
```sql
SELECT DATE(start_time) as date,
       COUNT(*) as count,
       SUM(total_volume) as volume
FROM trainings
WHERE start_time >= DATE_SUB(CURDATE(), INTERVAL 90 DAY)
GROUP BY DATE(start_time);
```

### 3. 训练频率趋势（折线图）
- **X轴**: 时间
- **Y轴**: 训练次数
- **切换**: 日/周/月视图
- **对比线**: 显示上周/上月同期数据

### 4. 训练量趋势（折线图）
- **X轴**: 时间
- **Y轴**: 总重量（kg）
- **切换**: 日/周/月视图
- **渐进超负荷**：高亮显示进步趋势

### 5. 部位统计

#### 5.1 各部位训练次数（柱状图）
```sql
SELECT e.category, COUNT(DISTINCT ts.training_id) as count
FROM training_sets ts
JOIN exercises e ON ts.exercise_id = e.id
WHERE ts.created_at >= DATE_SUB(CURDATE(), INTERVAL 30 DAY)
GROUP BY e.category
ORDER BY count DESC;
```

#### 5.2 部位均衡度（雷达图）🎯
- 显示各部位训练均衡度
- 帮助发现训练偏科
- 参考训记app的雷达图设计

#### 5.3 部位训练量分布（饼图）
- 显示各部位训练量占比

### 6. 动作排名（列表）
- Top 10 训练量最大的动作
- Top 10 最常训练的动作
- 点击可查看动作历史记录

### 7. 个人记录（PR）🏆
- 各动作最大重量
- 各动作最大训练量（重量×次数）
- PR 达成时间
- PR 冲击提醒（接近记录时高亮）

### 8. 渐进超负荷追踪 📈
- 同动作历史训练量对比
- 显示进步/停滞/退步趋势
- 瓶颈动作识别

### 9. 周/月对比
- 本周 vs 上周：训练次数、训练量、时长
- 本月 vs 上月：训练次数、训练量、时长
- 进步百分比显示

### 10. 身体数据趋势
- 体重变化曲线
- 体脂率变化曲线
- 支持选择时间范围：1个月/3个月/6个月/1年

## 四、API 设计

### 4.1 总览统计
```
GET /api/stats/overview?period=30
```

### 4.2 训练日历
```
GET /api/stats/calendar?days=90
```

### 4.3 训练频率趋势
```
GET /api/stats/frequency?period=30&granularity=day
```

### 4.4 训练量趋势
```
GET /api/stats/volume-trend?period=30&granularity=day
```

### 4.5 部位统计
```
GET /api/stats/body-parts?period=30
```

### 4.6 动作排名
```
GET /api/stats/exercise-ranking?period=30&limit=10
```

### 4.7 周/月对比
```
GET /api/stats/comparison?type=week
```

### 4.8 渐进超负荷
```
GET /api/stats/progression?exercise_id=123&days=90
```
